# CI Скрипты для Continuous Integration

Проект содержит два альтернативных CI скрипта для выполнения непрерывной интеграции.

## Доступные скрипты

### 1. Bash скрипт (`ci.sh`)
- **Файл**: `ci.sh`
- **Запуск**: 
  - Через Git Bash: `bash ci.sh`
  - Через WSL: `wsl bash ci.sh`
  - Через bat-файл: `run_ci.bat` (двойной клик)

### 2. PowerShell скрипт (`ci.ps1`)
- **Файл**: `ci.ps1`
- **Запуск**:
  - Через PowerShell: `powershell.exe -ExecutionPolicy Bypass -File ci.ps1`
  - Через bat-файл: `run_ci_ps1.bat` (двойной клик)

## Что выполняют скрипты

Оба скрипта выполняют одинаковые операции:

1. ✅ **Загрузка актуального состояния с сервера** - `git pull`
2. ✅ **Сборка проекта** - копирование файлов в директорию `build/`
3. ✅ **Сборка unittest** - установка зависимостей через `npm install`
4. ✅ **Выполнение unittest** - запуск тестов через `npm test`
5. ✅ **Создание установщика** - генерация MSI файла через WiX Toolset
6. ✅ **Установка приложения** - автоматическая установка MSI через msiexec

## Требования

- Git (для обновления кода)
- Node.js и npm (для тестов)
- WiX Toolset v3.14 (для создания MSI установщика)
- PowerShell (для PowerShell скрипта) или Git Bash/WSL (для bash скрипта)

## Особенности

- **Окно не закрывается автоматически** - после выполнения скрипт ждет нажатия Enter
- **Подробный вывод** - каждый шаг отображается с цветной индикацией
- **Обработка ошибок** - при ошибке скрипт останавливается с понятным сообщением
- **Автоматический поиск WiX** - скрипты ищут WiX Toolset в стандартных местах установки

## Рекомендации

- Для Windows рекомендуется использовать **PowerShell скрипт** (`ci.ps1`) - он нативный для Windows
- Для Linux/WSL рекомендуется использовать **Bash скрипт** (`ci.sh`)
